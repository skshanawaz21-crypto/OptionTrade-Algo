# Docs

This folder is for durable project documentation that is more detailed than the
top-level status files.

## Suggested Use

- Put strategy writeups here once rules are stable
- Add broker or deployment runbooks here
- Keep architecture notes here when they outgrow `DECISIONS.md`
- Store troubleshooting guides for repeat operational issues

## Current Docs

- `strategy-spec.md`: working template for the actual trading strategy

## Relationship To Top-Level Files

- `PROJECT_STATUS.md`: quick snapshot of where the project stands
- `ROADMAP.md`: phase-level delivery plan
- `NEXT_STEPS.md`: immediate resume checklist
- `DECISIONS.md`: decision log
- `TASKS.md`: practical execution tracker
- `CHANGELOG.md`: dated project history
